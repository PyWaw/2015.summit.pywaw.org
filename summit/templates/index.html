About

PyWaw Summit is a conference organized for everyone interested in Python-related technology, taking place in Warsaw, Poland on 25-26th of May, 2015.

Two days of great talks in English led by mix of local and international speakers. The aim is to provide valuable content in a friendly atmosphere that favours experience sharing and fun - please check out our code of conduct.

The conference derives from experienced local Python community called PyWaw, which actively organizes monthly meetups for over 3.5 years. There were already more than 45 meetings and over 100 presentations.

Willing our community to grow, PyWaw decided to go one step beyond and organize a conference that makes a dent in the Python universe.

    Concepts for designing large and scalable Python applications that work in practice. Python is often referred to as a scripting language. He is the primary author of Velociraptor, an open source platform-as-a-service for deploying Heroku-compatible applications, and of Mettle, a data pipeline framework being open sourced at the PyWaw conference.

    Talk: Real-time web apps with (just) Python and Postgres

    The design and documentation of the leading Python web frameworks are heavily focused on building web sites, with HTML content rendered on the server, and require significant effort to support modern web apps that use RESTful APIs, real-time updates over websockets, and Javascript-based user interfaces.

    If we start with a clean slate and design a framework around the needs of a real-time single-page-application, several things become apparent:

    1. A big framework like Django has a lot of stuff we don't need, while still missing a lot of stuff we do need.
    2. Minimalist Python HTTP libraries (Werkzeug, CherryPy) can still do most of the heavy lifting on the server side, and save us from having to start completely from scratch.
    3. Gevent can allow a Python application to scale to large numbers of simultaneous connections without littering a codebase with hard-to-follow callbacks.
    4. Without a big framework in the way, an API can cleanly add live websocket streams to existing RESTful endpoints, to deliver exactly the data that clients are interested in.

    Using the well-known TodoMVC application as an example, this presentation will give a tour of a simple Python backend that provides live syncing between browsers using just Python, Postgres, and a Werkzeug based micro framework that integrates REST endpoints and websockets into a single, coherent API.

    Though approachable to beginners, this talk will be of greatest interest to intermediate-level Python programmers who have used Django in the past and wondered how to move on to something more modern, but not more complicated.

    I've been using Python profesionally since 2009. I'm a developer and an Open Source contributor for a number of projects.     Talk: On snakes and elephants

    Everyone knows you can connect to PostgreSQL from Python, but not everyone knows you can actually make the PostgreSQL server process execute Python code in an embedded interpreter.

    PL/Python is a stored procedure language for PostgreSQL, allowing extending Postgres with Python.

    The use cases are limitless, from moving your business logic into the database, to doing number crunching inside the database server process without having to ship raw data to external worker nodes.

    We'll talk about how would you write an embedded Python procedure inside Postgres, what functionality it offers and what are the quirks and challenges posed by running a Python interpreter inside a database server process.
